# Teacher-Label State25 마스터 플랜

## 목적

이 문서는 `1분봉 25개 teacher-state 최종판`을 실제 시스템에 편입하기 위한 상위 기준 문서다.

지금 해야 할 일은 새 장세 체계를 처음부터 다시 만드는 것이 아니라,

- 이미 구축된 `position -> state -> forecast -> decision -> wait -> exit -> close result`
- 그리고 이미 존재하는 response/state 재료

위에 `teacher-state 25`와 `micro-structure canonical state`를 얹어, 사람이 보는 차트 패턴과 시스템이 배우는 상태를 더 잘 연결하는 것이다.

## 현재 판단

### 이미 충분히 강한 것

- 시장 분위기 / regime
- 구조 위치 / response
- 진입/대기/청산 해석
- close 결과 라벨

즉 지금 시스템은 이미:

- 지금 장이 거친지 조용한지
- 지금 가격이 상단/하단/중앙 어디인지
- 지금은 observe인지 blocked인지 wait인지
- 기다리면 더 좋은지, 빨리 자르는 게 좋은지

를 꽤 잘 알고 있다.

### 아직 약한 것

부족한 것은 대체로 `차트 모양 자체를 직접 수치화한 층`이다.

대표적으로:

- 몸통 평균
- 윗꼬리/아랫꼬리 비율
- 도지 비율
- 연속봉
- 압축 비율
- 거래량 burst 후 decay
- 고점/저점 재시험 횟수
- 갭 메움 진행률

같은 필드가 필요하다.

## 구현 방향

핵심 원칙은 이렇다.

1. 새 데이터를 마구 더 받는 게 아니라 `기존 1분봉 OHLCV`를 재가공한다.
2. response 안에 이미 있는 좋은 재료는 버리지 않고 재사용한다.
3. 하지만 최종 학습/판단용 공용 필드는 `state` 쪽에 canonical하게 둔다.
4. `entry_decisions`와 `closed history`에는 필요한 compact surface만 export한다.

## 결론

지금 `state25` 구현은 새 체계를 새로 만드는 작업이 아니라,

- 이미 잘 만들어진 response/state/forecast 체계 위에
- 차트 모양을 직접 표현하는 micro-structure 필드를 canonical state로 승격해
- 사람 기준 teacher-state와 시스템 기준 state를 더 잘 이어붙이는 작업

으로 보는 게 맞다.
