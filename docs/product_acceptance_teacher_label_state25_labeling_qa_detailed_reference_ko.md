# Teacher-Label State25 Labeling QA 상세 기준서

## 목적

이 문서는 `teacher-state 25`를 compact dataset에 붙일 때 필요한 `검수 / 누수 방지 / 혼동 관리 / 희소 패턴 감시` 기준을 고정한다.

이번 문서의 목표는 간단하다.

- 라벨링 품질을 학습 전에 먼저 안정화한다.
- 학습이 알아서 해결해주지 않는 위험 요소를 미리 막는다.
- 반영할 규칙과, 나중 실험으로 조정할 항목을 분리한다.

## 지금 바로 고정할 운영 규칙

### 1. Look-ahead bias 금지

이 항목은 실험으로 미룰 수 없다. 반드시 지금 고정해야 한다.

금지 규칙:

- `Rcur` 계산 시 현재 의사결정 시점 이후 봉 정보를 섞지 않는다.
- `Hret`, `Lret` 계산 시 미래 고점/저점을 사용하지 않는다.
- `VB`, `VD` 계산 시 현재 판단 시점 이후 거래량을 사용하지 않는다.
- `teacher-pattern` 라벨은 항상 해당 시점까지 관찰 가능한 20~50봉만 기준으로 붙인다.

원칙:

- `entry label`은 entry 시점까지의 정보만
- `wait label`은 wait decision 시점까지의 정보만
- `exit label`은 exit decision 시점까지의 정보만 사용한다

### 2. 2단계 라벨링 검수 프로세스

라벨링은 한번 붙이고 끝내지 않는다.

1차:

- 라벨러가 10K를 붙인다.
- 리뷰어가 전체 중 `20% 샘플`을 재라벨링한다.

2차:

- `confidence가 낮은 상위 5%`만 별도 모은다.
- 이 구간만 관리자/최종 검수자가 본다.

목적:

- 전체를 전부 다시 보는 비효율을 피한다.
- 대신 애매한 케이스에 검수 시간을 집중한다.

### 3. confusion pair watchlist 고정

아래 패턴 쌍은 사전 혼동 주시 대상으로 둔다.

- `12 브레이크아웃 직전` ↔ `23 삼각수렴 압축`
- `5 Range 반전장` ↔ `10 공허한 횡보장`
- `16 페이크아웃 반전` ↔ `2 변동성 큰 장`

운영 규칙:

- 라벨링 검수 문서에 이 3쌍을 따로 기록한다.
- 첫 10K 결과에서는 전체 confusion보다 이 3쌍 confusion을 우선 본다.

### 4. 희소 패턴 감시

아래 패턴은 희소 클래스가 될 가능성이 높다.

- `3 갑자기 발작장`
- `17 거래량 폭발장`
- `19 속도감 추세장`

운영 규칙:

- 첫 10K 라벨링 후 발생률을 확인한다.
- `1% 미만`이면 경고로 표시한다.
- 단, 희소하다고 해서 즉시 병합하거나 삭제하지 않는다.

### 5. 라벨링 후 필수 검증 체크

10K 라벨링 직후 아래를 꼭 본다.

1. 패턴 25개별 발생률
2. `primary + secondary` 조합 빈도
3. `entry / wait / exit` 행동 bias 분포
4. 자산별 패턴 분포
5. confusion pair 상위 쌍

권장 기준:

- `1% 미만` 패턴은 경고
- `12+23` 같은 주요 보조패턴 조합은 과도하게 몰리지 않는지 확인
- `entry:wait:exit`가 한쪽으로 지나치게 쏠리지 않는지 확인

## 지금은 고정하지 않고 실험으로 둘 항목

아래는 지금 문서로 고정하지 않고, 1차 라벨링 결과를 본 뒤 조정하는 게 맞다.

### 1. 자산별 ATR 실제 분포

외부 조언에서 제시한 BTC/XAU/NAS ATR 예상치는 참고값으로만 둔다.

운영 원칙:

- 첫 `1K` 라벨링 후 우리 데이터 기준으로 다시 계산한다.
- 그 뒤 `ATRP_20`, `ATRP_10` 기준을 자산별로 보정한다.

### 2. 학습 split 비율

`70 / 15 / 15`는 나쁘지 않지만 지금 확정할 필요는 없다.

운영 원칙:

- baseline 학습 직전 실제 데이터 크기와 자산별 분포를 본 뒤 결정한다.
- 패턴 희소성이 심하면 stratified split이 우선이다.

### 3. baseline 모델과 목표 수치

`XGBoost baseline`, `F1 > 0.65`는 참고 목표로 둔다.

다만 확정 규칙은 아니다.

운영 원칙:

- `macro F1`
- 패턴별 recall
- confusion 상위 쌍 precision/recall

을 같이 본다.

즉 `F1 하나`로 execution 여부를 결정하지 않는다.

### 4. execution 반영 조건

아래는 지금 고정하지 않는다.

- `F1 몇 이상이면 실행 반영`
- `top3 confusion만 고치면 된다`

이건 baseline 결과를 본 뒤 정하는 게 맞다.

## 검수 문서가 필요한 이유

학습이 알아서 해결해줄 것처럼 보이는 항목도 있지만, 실제로는 다르다.

- look-ahead bias는 학습이 고쳐주지 않는다.
- 애매한 패턴 간 경계는 라벨 품질이 무너지면 모델도 그대로 무너진다.
- 희소 패턴은 학습 전에 존재 여부부터 확인해야 한다.

즉 이 문서는 학습 전에 품질 바닥을 잡아주는 역할이다.

## 결론

지금 바로 문서에 반영할 것은 아래다.

- look-ahead bias 금지
- 2단계 라벨링 검수 프로세스
- confusion pair watchlist
- 희소 패턴 경고
- 라벨링 후 검증 체크리스트

반대로 아래는 실험으로 둔다.

- 자산별 ATR 실제 캘리브레이션
- split 비율
- baseline 모델 목표치
- execution 반영 조건

즉 `라벨 품질 관리`는 지금 고정하고, `모델/실험 세팅`은 뒤로 미루는 것이 맞다.
