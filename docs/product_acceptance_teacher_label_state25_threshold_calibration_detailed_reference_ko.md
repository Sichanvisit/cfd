# Teacher-Label State25 Threshold Calibration 상세 기준서

## 목적

이 문서는 `teacher-state 25`를 실제 compact dataset row에 붙일 때 사용할 `정량 임계치 v2` 기준을 정리한다.

핵심 원칙은 단순하다.

- 기존 고정 % 기준은 완전히 버리지 않는다.
- 하지만 BTC / XAU / NAS가 같은 절대 %를 같은 의미로 쓰지 않으므로 `ATR 정규화가 섞인 hybrid 기준`으로 옮긴다.
- 이 기준은 먼저 `teacher-pattern 라벨링`에 적용하고, 바로 실전 실행 룰로 강제하지 않는다.

## 왜 보정이 필요한가

기존 25개 표는 사람 기준 teacher-state로는 충분히 좋다.
하지만 정량 라벨러로 내려오면 아래 문제가 생긴다.

1. `BTC 1%`와 `XAU 1%`는 의미가 다르다.
2. `도지 비율`, `거래량 burst`, `압축 비율`은 패턴마다 요구 강도가 달라야 한다.
3. `브레이크아웃 직전`, `삼각수렴`, `더블탑/바텀`은 고정 절대치보다 구조 확인 강도가 더 중요하다.

즉 패턴 이름은 유지하되, 수치 기준은 `고정 % + 상대 변동성`으로 보정하는 편이 맞다.

## 최종 채택 원칙

### 1. 그대로 유지하거나 거의 유지

- `1 쉬운 루즈장`
- `10 공허한 횡보장`
- `4 추세 지속장`

이 계열은 기존 기준이 너무 공격적이지 않았고, 사람 해석과도 크게 어긋나지 않는다.

### 2. ATR 정규화가 필요한 패턴

- `2 변동성 큰 장`
- `3 갑자기 발작장`
- `14 모닝 컨솔리데이션`
- `19 속도감 추세장`

이 패턴은 자산별 체감 차이가 커서 절대 %만 쓰면 왜곡이 심하다.

### 3. 구조 확인을 더 엄격하게 해야 하는 패턴

- `5 Range 반전장`
- `12 브레이크아웃 직전`
- `15 캔들 연속 패턴`
- `23 삼각수렴 압축`

이 패턴은 그냥 “비슷해 보이는 모양”으로는 부족하고, 재시험 횟수나 압축 강도를 더 엄격히 봐야 한다.

### 4. 거래량은 2단계로 보는 편이 좋다

기존 `VB >= 1.3`, `VB >= 2.5` 같은 단일 기준은 너무 납작하다.
앞으로는 아래처럼 나눈다.

- `meaningful volume increase`: `VB >= 1.8`
- `strong burst`: `VB >= 2.5`
- `explosive burst`: `VB >= 3.5`

즉 `17 거래량 폭발장`은 기본적으로 `3.5`에 가깝게 보고,
`12 브레이크아웃 직전`이나 `15 캔들 연속 패턴`은 `1.8` 또는 `2.5`만으로도 충분할 수 있다.

다만 후속 교차검토를 반영해, `17 거래량 폭발장`의 실전 라벨링 기준은 아래처럼 한 단계 조정한다.

- `explosive burst`: `VB >= 3.0`
- 단, 가능하면 `burst 지속 3봉` 또는 그에 준하는 지속 확인이 붙어야 한다

현재 시스템에 `burst 지속 3봉` 전용 필드가 없다면, 초기 v2 라벨러에서는 `VB >= 3.0 + 낮은 VD(decay 완만)`를 proxy로 쓴다.

## 공통 정량 변수

다른 모델이나 후속 라벨러가 이해하기 쉽게 아래 공통 변수를 쓴다.

- `ATRP_20`: `ATR_20 / close * 100`
- `ATRP_10`: `ATR_10 / close * 100`
- `B`: 최근 20봉 평균 몸통 비율
- `UW`: 최근 20봉 평균 윗꼬리 비율
- `LW`: 최근 20봉 평균 아랫꼬리 비율
- `D`: 최근 20봉 도지 비율
- `Rcur`: 현재 같은 방향 연속봉 개수
- `Rmax`: 최근 20봉 최대 연속봉 개수
- `BullShare`: 최근 20봉 양봉 비율
- `C`: 범위 압축 비율
- `VB`: 최근 20봉 거래량 burst 비율
- `VD`: burst 이후 거래량 decay 비율
- `Hret`: 최근 20봉 비슷한 고점 재시험 횟수
- `Lret`: 최근 20봉 비슷한 저점 재시험 횟수
- `GF`: 갭 메움 진행률

주의:

- 현재 시스템에 `ATRP_20`, `ATRP_10`가 바로 hot surface로 있지 않다면, 초기 라벨러에서는 `regime_volatility_ratio`를 임시 proxy로 쓸 수 있다.
- 하지만 정식 v2 라벨러로 가려면 `ATRP helper`는 추가하는 편이 좋다.

## 패턴별 v2 보정안

| ID | 패턴 | 기존 해석 | v2 보정안 | 판단 |
|---|---|---|---|---|
| 1 | 쉬운 루즈장 | 고정 낮은 몸통 + 높은 도지 비율 | `B <= min(0.10%, 0.6*ATRP_20)`, `D >= 0.25`, `VB < 1.5` | 채택 |
| 2 | 변동성 큰 장 | 절대 몸통 크기 중심 | `B >= 1.5*ATRP_20`, `UW/LW`도 높음, `VB >= 1.8` | 채택 |
| 3 | 갑자기 발작장 | 스파이크 % 중심 | `single spike >= 2.5*ATRP_10`, `VB >= 3.5` | 채택 |
| 4 | 추세 지속장 | BullShare 우위 | `BullShare >= 0.80` 유지 | 유지 |
| 5 | Range 반전장 | 재시험 2회 | `Hret/Lret >= 2`, `D >= 0.30`, `FalseBreakRisk` 보조 | 수정 채택 |
| 12 | 브레이크아웃 직전 | 압축 0.55 | `C >= 0.70`, `VB >= 1.8`, `B` 매우 낮음 | 채택 |
| 14 | 모닝 컨솔리데이션 | 범위 0.20% | `range <= 0.8*ATRP_20`, `D >= 0.25`, `C >= 0.40` | 채택 |
| 15 | 캔들 연속 패턴 | 연속봉 5개 | `Rcur >= 5` or (`Rcur >= 4` and `VB >= 2.0`) | 수정 채택 |
| 17 | 거래량 폭발장 | `VB >= 2.5` | `VB >= 3.0` and `burst 지속 3봉` 우선, 초기 라벨러는 `VB >= 3.0 + 낮은 VD` proxy 허용 | 수정 채택 |
| 19 | 속도감 추세장 | 15분 6% | `누적 이동 >= 3*ATRP_20`, `B` 크고 `VD` 낮음 | 채택 |
| 23 | 삼각수렴 압축 | `C >= 0.60` | `C >= 0.75` and `Hret >= 2` and `Lret >= 2` | 채택 |

## 전역 보정 규칙

### 도지 비율

- 예전처럼 `D >= 0.35`를 넓게 쓰면 1분봉에서는 너무 빡빡할 수 있다.
- v2에서는 `D >= 0.25`를 기본선으로 두고,
- 정말 공허한 횡보나 엔진 꺼짐 같은 패턴만 `D >= 0.30 ~ 0.50`을 쓴다.

### 거래량 burst

- `1.3`은 의미 없는 잡음이 섞일 가능성이 높다.
- 최소 `1.8`부터 “의미 있는 증가”로 보고,
- `3.0`은 “폭발”의 실무 기준으로 본다.
- 다만 가능하면 `지속 3봉` 같은 지속 확인을 붙인다.
- 지속 필드가 없을 때만 `낮은 VD`를 임시 proxy로 쓴다.

### 압축 비율

- `브레이크아웃 직전`, `삼각수렴` 같은 압축 패턴은 v1보다 더 보수적으로 잡는 편이 좋다.
- v2에서는 `0.70`, `0.75`처럼 더 높은 압축 강도를 요구한다.

### 연속봉

- 연속봉 개수만 보고 진입형 패턴으로 붙이면 가짜 모멘텀을 과대평가할 수 있다.
- 따라서 `Rcur >= 5`를 기본선으로 두고,
- 한 단계 완화가 필요할 때만 `Rcur >= 4 + VB >= 2.0` 조합을 쓴다.

### Range 재시험

- `Range 반전장`은 `재시험 3회`가 너무 빡빡할 수 있다.
- 1분봉에서는 `재시험 2회 + D >= 0.30 + false-break 위험 보조`가 더 현실적이다.

## 자산별 해석 원칙

정확한 수치는 학습/백테스트로 보정해야 하지만, 해석 원칙은 아래처럼 둔다.

| 자산 | 해석 원칙 |
|---|---|
| `BTC` | 절대 %가 크게 보여도 일상 변동일 수 있으니 ATR 정규화 비중을 높인다 |
| `XAU` | 절대 변동이 작아 보이더라도 구조적 전환이 빠를 수 있으니 wick/retest를 더 중요하게 본다 |
| `NAS` | session-driven 움직임이 강하므로 모닝/갭/급발산 패턴에서 세션 문맥을 더 강하게 본다 |

## 실전 적용 순서

1. 문서 기준은 v2로 갱신한다.
2. `teacher-pattern` 라벨러는 먼저 v2 기준으로 compact dataset에 붙인다.
3. 1주 정도 샘플링해서 패턴 hit-rate와 confusion을 본다.
4. 그 다음에야 자동 추천 라벨러나 실행 bias 보정으로 넘긴다.

즉 지금 단계에서는 `바로 실전 execution 룰`로 박는 게 아니라, `teacher-label quality 개선`이 우선이다.

## 결론

이번 외부 조언은 대체로 받아들이는 게 맞다.
다만 그대로 복붙하기보다 아래처럼 쓰는 것이 좋다.

- `고정 절대치만 쓰는 패턴` -> 유지 또는 약보정
- `자산별 왜곡이 큰 패턴` -> ATR 정규화로 변환
- `압축/재시험 패턴` -> 구조 확인을 더 엄격하게
- `거래량` -> 단계형 burst 체계 + 가능하면 지속 확인까지 같이 본다

추가 채택한 미세 조정은 아래와 같다.

- `5 Range 반전장` -> `재시험 2회 + 도지 강화`
- `15 캔들 연속 패턴` -> `연속 5봉 기본 / 4봉이면 거래량 2.0 이상 요구`
- `17 거래량 폭발장` -> `VB 3.0 + 지속 3봉`, 초기에는 `낮은 VD` proxy 허용

이 방향으로 가면 state25는 사람 설명과 시스템 수치가 더 잘 맞는 쪽으로 정리된다.
